Refactoring components

What

A combination of the styles, components and guidance in response to a particular problem makes a design pattern.

In the service manual there are a number of components that could become design patterns and be built differently using a combination of existing styles and components.

Why

This work would:

  • reduce the size of the codebase (the NHS.UK frontend), making it easier to maintain
  • increase the versatility of the design system, with resuable styles and components
  • give users clearer context of usage, with design patterns

Note: It would be a breaking change, meaning users of the frontend would need to manually update their projects with the latest code, so it all needs to be considered carefully.

Components to refactor

  • Do and don't list
  • List panel
  • Review date
