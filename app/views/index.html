A combination of the styles, components and guidance in response to a particular problem makes a design pattern.

In the NHS.UK frontend and NHS digital service manual there are a number of components that should be design patterns, which could also be built differently using a combination of styles and components, without changing the look and feel.

The benefits of doing this include:

  • reducing the size of the codebase, making it easier to maintain
  • increasing the flexibility of the design system
  • giving clearer context of usage, by making design patterns

It would be a breaking change, meaning users of the frontend would need to manually update their projects with the latest code, so we need to take that into consideration.
