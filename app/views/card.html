New component Card

Overview

The panel and promo are two components in the NHS.UK frontend that do not have guidance in the service manual. They both also share a lot of the same structure and design. This has confused users of the design system, with people not knowing when and how to use the components.

The content of the current promo component is all contained within a link (anchor tag) and this can be a difficult experience for screenreader users.

We are proposing to combine the two components into one new component (card), add guidance to the service manual and improve the accessibility with new code.

Backlog issue 159 (GitHub)

Design decisions

We have added a:

  • border around the cards to stop them blending into the grey background
  • thicker bottom border and a darker border on hover for the clickable cards, to indicate the entire card can be selected

When to use a card

Use a card component to provide a brief summary of content or a task, often with a link to more detail.

Clickable card

When using the clickable card, you place a link within the card heading, and add the class nhsuk-card--clickable to the parent level nhsuk-card.

When this class is applied, it will trigger some JavaScript and users will be able to click the entire card.

If the JavaScript is not available the card will fallback to only the heading link being clickable.
