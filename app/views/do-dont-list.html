Proposal

We are proposing to:

  • rewrite the do and don't list component as a design pattern
  • remove the do and don't list code from the frontend and build the pattern using other styles and components

Pattern

The pattern could be something like "Help users understand what they should and shouldn't do" and built using the feature card component and ticks and crosses list style.

Other design patterns could use the same styles and components.
